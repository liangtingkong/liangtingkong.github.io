函数高级

原型与原型链

函数的prototype属性

  • 每个函数都有一个prototype属性，它默认指向一个object空对象（即为原型对象）
  • 原型对象有一个属性constructor，它指向函数对象

给原型对象添加属性（一般是方法）==>实例对象引用

显示原型和隐式原型

  • 每个函数都有一个prototype 即显式原型

  • 每个实例对象都有一个_proto_ （隐式原型）

  • 对象的隐式原型的值为其对应构造函数显示原型的值

原型链（查找对象的属性）
instanceof是如何判断的？
  • 表达式 A （实例对象）instanceof
  • 如果B函数的显示原型对象在A对象的原型链上 返回true 否则返回 false
  • Function是通过new自己产生的实例

执行上下文与执行上下文栈

变量声明提升

  • 通过var定义（声明）的变量，在定义语句之前就可以访问到
  • 值：undefined

函数声明提升

  • 通过function声明的函数，在之前就可以直接调用
  • 值：函数定义（对象）
执行上下文

  • 代码分类

    • 全局代码
    • 局部代码

  • 全局执行上下文

    • 在执行全局代码前将window确定为全局执行上下文‘
    • 对全局数据进行预处理
    • var定义的全局变量==>undefined ，添加为window的属性
    • function声明的全局函数==>赋值（fun)，添加为window的方法
    • 开始执行上下文

  • 函数执行上下文

    • 在调用函数，准备执行函数体之前，创建函数执行上下文对象
    • 对局部数据进行预处理
    • 形参变量==>赋值（实参）添加为执行上下文属性
    • arguments==>赋值（实参列表）
    • var 局部变量==>undefined
    • unction声明的局部函数==>赋值（fun)，
    • this==>赋值(调用函数对象)
    • 开始执行函数代码

  • 执行上下文栈

    • 全局代码执行前，js会创建一个栈来存储管理所有的执行上下文对象

    • 在全局执行上下文确定后，将其添加到栈中

    • 在函数执行上下文创建后，将其添加到栈中

    • 、当前函数执行完后将栈顶的对象移除栈

    • 代码执行完后，栈中只剩下window

作用域与作用域链

作用域

  • 代码所在的一块区域
  • 它是静态的，在编写代码是就确定了

分类

  • 全局作用域
  • 函数作用域
  • es6 有了块级作用域（let）

作用

  • 隔离变量，不同作用域下同名变量不会冲突

闭包

init() 创建了一个局部变量 name 和一个名为 displayName() 的函数。displayName() 是定义在 init() 里的内部函数，并且仅在 init() 函数体内可用。请注意，displayName() 没有自己的局部变量。然而，因为它可以访问到外部函数的变量，所以 displayName() 可以使用父函数 init() 中声明的变量 name 。

  • 闭包存在与嵌套的内部函数中
  • 产生闭包的条件
    • 函数嵌套
    • 内部函数引用了外部的数据

引用的外部数据回存在闭包中

（第一次引用内部函数）

（第二次引用，数据a的自增并没有消失，而是存在了闭包中）

闭包的作用
  1. 使用内部变量在函数执行后，仍存活在内存中（延长了局部变量的生命周期）
  2. 让函数外部可以操作到函数内部的数据
    • 函数执行完后，局部变量消失。存在于闭包中的标量有可能存在
  3. 在函数外部可以直接访问内部的局部变量吗
    • 不能，可以通过闭包让外部操作他
闭包的生命周期
闭包的应用

自定义js 模块

  • 具有特定功能的js文件

  • 将所有数据和功能都封装在函数内部

  • 只向外暴露一个包含 n个方法的对象或函数

  • 模块的使用者，只需要通过模块暴露的对象调用方法来实现对应的功能

闭包的缺点和解决

缺点

  • 函数执行完之后，函数内的局部变量没有释放，占用的内存时间回变长
  • 容易造成内存泄漏

解决

  • 能不用闭包就不用闭包
  • 及时释放

内存溢出

  • 程序运行的错误
  • 当程序运行需要的内存超过了剩余的内存是，就抛出了内存溢出的错误

内存泄漏

  • 占用的内存没有及时释放

  • 内存泄漏积累多了就用一导致内存溢出

  • 常见的内存溢出

    • 意外的全局变量
    • 没有及时清理计时器或回调函数
    • 闭包

面向对象高级

对象创建模式

  1. object构造函数模式

    • 套路：先创建object对象，在动态添加属性

    • 使用场景：开始有不确定对象内部数据

    • 问题：语句太多

    • ```js
      /人 ：name：tom ，age：12/

        var p=new Object()
  p.name="tom"
  p.age=12
  p.setName = function(name){
    this.name=name
  }

  2. 工厂模式

    • 套路：通过工厂函数动态创建对象并返回

    • 使用场景：需要创建多个对象

    • 问题：对象没有具体的类型，都是object

    • ```js

       function creatPerson(name,age){
    var obj={
      name:name,
      age:age,
      setName:function(name){
        this.name=name
      } 
    }
    return obj
  }
  var p=creatPerson("mqm",22)

  3. 构造函数+原型的组合模式

    • 套路：自定义构造函数，属性在函数中，方法添加到原型上

    • 适用场景：需要创建多个类型确定的对象

    • ```js
      function Person(name,age){

        this.name=name,
  this.age=age
}
Person.prototype.setName=function(name){
  this.name=name
}
var p=new Person('mqm',22)

  4. 借用构造函数继承(假的)

    1. 套路：

      1. 定义父函数构造函数
      2. 定义子函数构造函数
      3. 在子类型构造函数中调用父类型构造

    2. 关键：

      1. 在子类型构造函数中通用call（）调用父类型构造函数

    3. 代码

      • ```js
        function Person(name,age){
            this.name=name
    this.age=age
  }

  function Student(name,age,price){
    Person.call(this,name,age)//相当于this.Person（name，age）
    /*
    this.name=name
    this.age=age*/
    this.price=price
  }

  var s= new Student('mqm',22,22)
  console.log(s);

线程机制与事件机制

  1. 进程与线程

    1. 进程

      • 程序的一次执行，它占有一篇独有的内存空间
      • 通过windows任务管理器查看进程

    2. 线程

      • 是进程里的一个独立执行单元
      • 是程序执行的一个完整流程
      • 是cpu的最小单元

    3. 图

    4. 相关知识

      • 程序必须运行在某个进程的某个线程上
      • 一个程序中至少有一个运行的线程：主线程，进程启动后自动创建
      • 进程中可以同时运行多个线程，我们会说程序是多线程运行
      • 一个进程内的数据可以供其中多个线程直接共享
      • 多个进程之间的数据是不能直接共享的
      • 线程池：保存多个线程对象的容器，实现线程对象的反复利用

    5. 相关问题

      1. 多线程与多进程
        • 多进程：一个程序可以同时运行多个实例
        • 多线程：一个进程，同时多个线程
      2. 单线程与多线程
        • 多线程：
          • 优点：能有效提升cpu的利用率
          • 缺点：创建多线程开销、线程间切换开销、死锁与状态同步
        • 单线程
          • 优点：顺序变成简单易懂
          • 缺点：效率低
      3. 浏览器是多线程运行的的
      4. 浏览器的进程
        • 多进程：chrome，edge
        • 单进程：firfox，ie

  2. 浏览器内核

    1. 内核：支撑浏览器运行的最核心的程序

      • chrome，safari：webkit、现在Blink
      • firefox：Gecko
      • ie：Trident

    2. 内核由很多模块组成

      • 主线程

        • js引擎模块
        • html、css文档解析模块
        • DOM/CSS模块
        • 布局和渲染模块：负责页面的布局和效果的绘制
        • …..

      • 分线程

        • 定时器模块
        • DOM事件相应模块
        • 网络请求模块：ajax

  3. js引擎执行代码的基本流程

    • 先执行初始化代码：包含一些特别的代码
      • 设置定时器
      • 绑定监听
      • 发送ajax请求
    • 后面在某个时刻才会执行回调代码
    • 为什么js要用单线程模式，而不是用多线程模式
      • js作为浏览器脚本语言，js的主要用途是与用户互动，以及操作dom
      • 这决定了它只能是单线程，否则会带来很复杂的同步问题
